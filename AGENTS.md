# Agent Instructions

## Workflow
- **Issue & Milestone Tracking**: Always read open issues and milestones in GitHub to understand the current priorities and requirements.
- **Development**: Develop code based on the linked GitHub issues.
- **Verification**: Add comprehensive tests that verify the functionality of any new feature or bug fix.
- **State Management**: Keep GitHub issues and milestones up to date as work progresses.
- **Deployment**: Commit and push completed code to the `development` branch.
